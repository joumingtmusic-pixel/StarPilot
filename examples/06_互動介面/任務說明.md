## 🎯 任務目標

學習如何用自然語言讓 AI 幫你建立美觀、易用的網頁介面，讓工具變得更友善。

---

## 💡 為什麼需要互動介面？

### 原本的使用方式（命令列）
```
👤：執行 python analyze.py --file data.csv --output report.xlsx
    ❌ 要記住指令格式
    ❌ 打錯一個字就失敗
    ❌ 不方便分享給同事
```

### 有了互動介面之後
```
👤：開啟網頁 → 上傳檔案 → 點擊按鈕 → 下載結果
    ✅ 直覺易用，不需要技術背景
    ✅ 視覺化呈現結果
    ✅ 分享網址就能讓同事使用
```

## 💡 提示

### 介面設計的黃金原則

**1. 簡潔明瞭**
```
✅ 好的設計：
- 重要資訊放最上面
- 按鈕文字清楚（「上傳檔案」比「確定」好）
- 適當的留白

❌ 不好的設計：
- 把所有功能塞在一起
- 按鈕文字模糊
- 擁擠的版面
```

**2. 即時回饋**
```
✅ 好的體驗：
- 上傳檔案時顯示進度條
- 處理中顯示「載入中...」
- 完成時顯示成功訊息

❌ 不好的體驗：
- 點擊按鈕後沒反應
- 不知道系統在做什麼
- 錯誤時沒有提示
```

**3. 錯誤處理**
```
✅ 友善的錯誤訊息：
「檔案格式錯誤，請上傳 CSV 或 Excel 檔案」

❌ 生硬的錯誤訊息：
「ValueError: invalid file format」
```

## 🎨 Streamlit 常用元件

### 輸入元件
```
- text_input()     # 單行文字輸入
- text_area()      # 多行文字輸入
- number_input()   # 數字輸入
- date_input()     # 日期選擇
- time_input()     # 時間選擇
- file_uploader()  # 檔案上傳
- camera_input()   # 相機拍照
```

### 選擇元件
```
- selectbox()      # 下拉選單（單選）
- multiselect()    # 多選框
- radio()          # 單選按鈕
- checkbox()       # 勾選框
- slider()         # 滑桿
```

### 顯示元件
```
- write()          # 萬用顯示
- dataframe()      # 互動式表格
- table()          # 靜態表格
- metric()         # 指標卡片
- json()           # JSON 顯示
- code()           # 程式碼區塊
```

### 圖表元件
```
- line_chart()     # 折線圖
- bar_chart()      # 長條圖
- area_chart()     # 面積圖
- pyplot()         # Matplotlib 圖表
- plotly_chart()   # Plotly 互動圖表
```

### 版面配置
```
- columns()        # 多欄布局
- tabs()           # 分頁標籤
- expander()       # 可展開區塊
- container()      # 容器
- sidebar          # 側邊欄
```

## 🚀 進階功能

### 1. 即時資料更新
```
👤：我想每 5 秒自動重新載入資料
    不需要手動重新整理

🤖：我會使用 st.rerun() 
    配合時間檢查來實現自動更新
```

### 2. 會話狀態管理
```
👤：使用者在不同頁面間切換時
    之前上傳的資料要保留

🤖：我會使用 st.session_state
    將資料儲存在會話中
```

### 3. 快取機制
```
👤：資料處理很慢
    可以快取結果嗎？

🤖：沒問題！我會用 @st.cache_data
    處理過的資料會被快取
    下次載入會更快
```

```
👤：在手機上看會不會跑版？

🤖：Streamlit 預設就是響應式設計
    會自動適應不同螢幕大小
```
